Cloud Resource Optimization and RBAC Streamlining

Improving efficiency and governance in large-scale Azure cloud environments at PepsiCo GBS through automation.

Jump to Impact Metrics

Problem Statement

The enterprise Azure environment suffered from resource sprawl and complex, inherited Role-Based Access Control (RBAC) assignments. This led to deployment failures due to excessive RBAC entries exceeding cloud scope limits, necessitating a complex manual cleanup process.

Aim and Proposed Work

The goal was to introduce automated governance to streamline RBAC permissions, resolve critical deployment errors, and significantly boost the cloud operations team's efficiency without manual intervention.

  • Solution: Developed non-disruptive, advanced PowerShell scripts to audit Azure subscriptions.
  • Automation Goal: Programmatically identify and remove stale or redundant role assignments in a controlled manner.

Key Technical Details

  • RBAC Cleanup: The PowerShell scripts effectively cleaned the inherited access structure, which resolved critical scope limit issues and accelerated user onboarding and resource deployment.
  • DevOps Integration: Integrated the automated processes with Jenkins and Terraform to establish Continuous Integration/Continuous Deployment (CI/CD) pipelines, ensuring future deployments adhered to governance policies from the start.

Impact and Key Results

The automation project delivered significant, measurable benefits across the cloud operations team, establishing a clean and scalable governance model.

Operational Efficiency Gains
  • Efficiency Boost: Increased team efficiency in access management by 20%, freeing up engineering hours for higher-value tasks.
  • Critical Issue Resolution: Permanently resolved major Azure scope limit errors that were previously blocking mission-critical deployments.
  • Improved Governance: Established a clean, scalable RBAC governance model for future cloud expansion.
